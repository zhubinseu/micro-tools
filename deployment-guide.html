方案 A：Dashboard + GitHub 自动部署 ⭐ 推荐

连接 GitHub 仓库后，每次 git push 到 main 分支自动触发部署，无需本地配置任何凭证。

第 1 步：确保代码已推送到 GitHub

# 确认远程仓库
git remote -v
# origin  https://github.com/zhubinseu/micro-tools.git

# 确认最新代码已推送
git log origin/main --oneline -3

第 2 步：授权 Cloudflare 访问 GitHub 仓库

1
打开 https://github.com/settings/installations
找到 Cloudflare Pages → 点击 Configure
2
在 Repository access 中选择 Only select repositories
从下拉列表中添加 micro-tools 仓库，点击 Save
⚠️ 常见坑：如果在 Cloudflare 中看不到仓库，99% 是这一步没配。GitHub App 默认不授权新仓库，必须手动添加。

第 3 步：在 Cloudflare 创建 Pages 项目

1
打开 Cloudflare Dashboard → 左侧 Workers & Pages
2
点击 Create → Pages → Connect to Git
3
授权 GitHub → 选择 zhubinseu/micro-tools 仓库 → 点击 Begin setup

第 4 步：配置构建设置（关键！）

Project name
micro-tools
Production branch
main
Framework preset
None
Build command
npm run build
Build output directory
out
Root directory
/
🚨 最大避坑点：Framework preset 必须选 None！

如果选了 Next.js，Cloudflare 会强制要求填写 Deploy command 和 Version command（且无法删除），导致 wrangler deploy 报错 "Workers-specific command in a Pages project"。

如果已经选了 Next.js 且无法删除 Deploy command：把 Framework preset 改为 None，两个字段就会自动消失或变为可选。

第 5 步：配置环境变量（可选）

在 Environment variables 区域添加：

变量名值说明
NODE_VERSION20指定 Node.js 版本

第 6 步：部署

点击 Save and Deploy，等待构建完成（约 1-2 分钟）。

✅ 部署成功后，你会获得一个 https://micro-tools.pages.dev 地址。之后每次 git push origin main 都会自动触发重新部署。

方案 B：Wrangler CLI 手动部署 终端操作

适合不想连接 GitHub、或需要本地控制部署时机的场景。

第 1 步：创建 Cloudflare API Token

1
打开 https://dash.cloudflare.com/profile/api-tokens
2
点击 Create Token → 选择模板 Edit Cloudflare Workers
3
确认权限包含：Account - Pages - Edit → 创建并复制 Token
🚨 Token 权限必须包含 Pages - Edit，否则部署时会报 Authentication error [code: 10000]。

第 2 步：本地构建

# 进入项目目录
cd /path/to/micro-tools

# 安装依赖（首次）
npm install

# 静态导出构建，产物输出到 out/
npm run build

第 3 步：部署到 Cloudflare Pages

# 设置 API Token 环境变量
export CLOUDFLARE_API_TOKEN="你的Token值"

# 部署（注意：必须用 pages deploy，不是 wrangler deploy！）
npx wrangler pages deploy out --project-name=micro-tools
⚠️ 命令区别：
✅ npx wrangler pages deploy out — Pages 项目部署
❌ npx wrangler deploy — Workers 项目部署，Pages 项目不能用！

或者直接用 package.json 中已配置好的命令：

export CLOUDFLARE_API_TOKEN="你的Token值"
npm run deploy  # 等价于 wrangler pages deploy out

首次部署会提示创建项目

如果项目不存在，Wrangler 会询问是否创建。输入 y 确认即可。

🌐 绑定自定义域名（可选）

1
在 Cloudflare Dashboard → 你的 Pages 项目 → Custom domains
2
点击 Set up a custom domain → 输入你的域名
3
如果域名在 Cloudflare DNS 管理，会自动添加 CNAME 记录；否则需手动添加： 
CNAME  www  micro-tools.pages.dev
4
等待 DNS 生效 + SSL 证书自动签发（通常几分钟内完成）

🔧 常见问题排查

问题 1：选不到 GitHub 仓库

原因：GitHub App 未授权该仓库

解决：访问 github.com/settings/installations → Cloudflare Pages → Configure → 添加仓库

问题 2："Workers-specific command in a Pages project"

原因：Framework preset 选了 Next.js，Deploy command 被强制设为 wrangler deploy

解决：Framework preset 改为 None，Deploy command 字段自动消失

问题 3："Authentication error [code: 10000]"

原因：API Token 缺少 Pages 写权限

解决：重新创建 Token，确保包含 Account - Pages - Edit 权限

问题 4：Deploy command / Version command 无法删除

原因：Framework preset 不是 None

解决：将 Framework preset 改为 None，两个字段变为可编辑或自动移除

问题 5：构建成功但页面 404

原因：Build output directory 配置错误

解决：确认设为 out（不是 dist / build / /）

问题 6：sitemap.xml 在开发模式报错

原因：Next.js 14 output: 'export' 与 Metadata Routes 在 dev 模式的已知 bug

解决：不影响生产环境。npm run build 生成的 out/sitemap.xml 完全正常

问题 7：本地 dev 服务器启动后端口被释放

原因：nohup npm run dev & 启动的进程随 shell 退出被回收

解决：使用后台任务方式启动，或直接在前端终端运行 npm run dev
